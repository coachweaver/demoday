The Mission

What you're building

Build an app that gives the user a smart response based on something they do or enter.

Pick your topic. Design your screen from scratch. Write the logic. The constraints are the engineering requirements, not the idea. Your app does one thing well.

The Standard

What every app must do

Required
Variables
At least one var that changes when the user interacts. A plain variable that stores and updates. No lists yet.
Required
onEvent Handlers
At least one onEvent() per interactive element. Each one updates a variable, then calls updateScreen().
Required
updateScreen Function
One updateScreen() function handles all visual changes. Every onEvent must call it. All setText calls live here.
Required
Concatenation
At least one output combines text and a variable: "Your result: " + result. On screen, not in the console.
Required
Conditional Logic
At least 3 if statements: minimum 2 full if/else blocks and 1 standalone if. Your app decides something.
Required
UI Built from Scratch
Design Mode only, no starter templates. Purpose should be obvious within the first 10 seconds.

Brainstorm Fuel

An app that ...

Starting points, not assignments. Change anything. Combine two. Make it yours.

Tells you what to eat

User picks mood, time of day, or hunger level. App recommends a meal or snack category.

Scores your day

User rates sleep, water, and activity. App gives a wellness score and a reaction.

Picks your outfit

User enters weather and occasion. App tells them what kind of outfit to wear.

Settles the debate

User answers 3 quick questions. App declares a winner. Great for school drama.

Reads your vibe

User clicks through mood questions. App returns a vibe of the day with a playlist genre.

Tells you if it's worth it

User enters a price and frequency of use. App gives a purchase verdict.

Ranks your weekend plan

User enters activities. App scores the plan on fun, cost, and effort.

Calculates the tip

User enters bill total and satisfaction level. App calculates the tip and reacts.

Decides if you need a jacket

User inputs a temperature. App responds with outfit advice, getting more sarcastic at extremes.

Builds a study plan

User inputs hours until a test and confidence level. App tells them where to focus.

Evaluation

How you'll be scored

Four categories. Three scores. A 2 is the goal -- it means your work meets the standard. A 3 means you went above and beyond. A 1 means something is missing and must be fixed.

Category 1Unacceptable 2Acceptable — the goal 3Exceptional
Functionality App crashes, freezes, or fails to respond to input as intended. App runs correctly. A user could pick it up and use it without guidance. App handles unexpected inputs gracefully with a noticeably polished experience.
Code Requirements One or more required elements are missing or non-functional. All required elements present and working: variable(s), onEvent, updateScreen(), concatenation, 3+ if statements. Code is clean, well-commented, and shows more sophisticated use of the required elements.
Design & Clarity UI is confusing or cluttered. Purpose unclear in the first 10 seconds. UI is purposeful. A new user understands what the app does without being told. Intentional visual choices meaningfully improve the user experience beyond the minimum.
Demo & Explanation Team cannot demo the app or explain what their code does. Team demos the app, states its purpose, and can explain a variable, an onEvent, and updateScreen(). Explanation shows why decisions were made, not just what the code does. Both partners speak clearly.
The standard is a 2. Most work should land here. A 3 is earned -- it requires clear extra craft or deeper thinking, and will not be given for simply completing the requirements. A 1 on any category must be revised and resubmitted until it reaches a 2. These four categories stay the same every sprint. Only the Code Requirements row updates as new concepts are added.
